
**Structure (top to bottom):**
- Outer container: `className="flex flex-col h-full bg-background border-r border-border"`
- Header: `<div className="p-4 border-b border-border"><h2 className="text-lg font-semibold text-foreground">Chat</h2></div>` (matches DesignEmptyState header pattern)
- Messages area: `<div className="flex-1 overflow-y-auto p-4 space-y-4">` with auto-scroll to bottom via `useRef` + `useEffect` on messages changes
  - For each message in `messages`:
    - User messages (role === 'user'): right-aligned bubble
      ```tsx
      <div key={msg.id} className="flex justify-end">
        <div className="max-w-[90%] rounded-lg px-4 py-2 bg-muted text-foreground">
          <div className="text-sm whitespace-pre-wrap break-words">{msg.content}</div>
        </div>
      </div>
      ```
    - Assistant messages (role === 'assistant'): left-aligned, full width
      ```tsx
      <div key={msg.id} className="flex justify-start">
        <div className="w-full text-sm text-foreground whitespace-pre-wrap">{msg.content}</div>
      </div>
      ```
  - After the first assistant message (opening statement), if `suggestedQuestions.length > 0`, render clickable chips:
    ```tsx
    <div className="flex flex-wrap gap-2 mt-2">
      {suggestedQuestions.map((q, i) => (
        <button key={i} type="button" onClick={() => sendMessage(q)}
          className="px-3 py-1.5 text-xs rounded-full border border-border bg-background hover:bg-muted transition-colors text-foreground">
          {q}
        </button>
      ))}
    </div>
    ```
    Suggestion chips shown only when `messages.length <= 1` (only opening statement exists) — disappear after first user message.
- Loading indicator: when `isLoading` and the last message is assistant with empty content, show a simple pulsing dots indicator: `<div className="flex gap-1"><span className="w-2 h-2 bg-muted-foreground/50 rounded-full animate-pulse" />` (3 dots with staggered animation-delay)
- Input area at bottom: `<div className="p-4 border-t border-border">`
  - Form with pill-shaped input per existing pattern:
    ```tsx
    <form onSubmit={handleSubmit} className="relative border border-border rounded-[24px] bg-background focus-within:ring-2 focus-within:ring-ring transition-shadow">
      <div className="flex items-end gap-2 px-3 py-2">
        <textarea value={input} onChange={(e) => setInput(e.target.value)}
          onKeyDown={(e) => { if (e.key === 'Enter' && !e.shiftKey) { e.preventDefault(); if (input.trim() && !isLoading) sendMessage(); } }}
          placeholder="Tell me about the email you'd like to create..."
          disabled={isLoading} rows={1}
          className="flex-1 border-0 bg-transparent focus-visible:ring-0 shadow-none px-2 py-2 text-sm resize-none outline-none min-h-[24px] max-h-[72px] overflow-y-auto" />
        <Button type="submit" disabled={isLoading || !input.trim()} size="icon"
          className="flex-shrink-0 h-9 w-9 rounded-full bg-muted hover:bg-muted/80 text-foreground disabled:opacity-50">
          <ArrowUp className="h-4 w-4" />
        </Button>
      </div>
    </form>
    ```

Create `app/brief/page.tsx` — the brief page that wires the hook to the UI components. This is a `'use client'` page component.

**Implementation:**

```tsx
'use client';

import { useRouter } from 'next/navigation';
import { useBriefChat } from '@/hooks/use-brief-chat';
import { BriefChatPanel } from '@/components/brief/brief-chat-panel';
import { BriefEmptyState } from '@/components/brief/brief-empty-state';

export default function BriefPage() {
  const router = useRouter();
  const { messages, suggestedQuestions, input, setInput, sendMessage, isLoading, isBriefComplete } = useBriefChat();

  const handleStartEditing = () => {
    router.push('/');  // Navigate to editor — per D-12, direct navigation, no confirmation
  };

  return (
    <div className="h-screen w-screen overflow-hidden flex flex-col">
      {/* Minimal header */}
      <header className="h-12 border-b border-border bg-background flex items-center px-6 flex-shrink-0">
        <h1 className="text-lg font-semibold text-foreground">Email Brief</h1>
      </header>

      {/* Two-panel layout */}
      <div className="flex flex-1 overflow-hidden">
        {/* Left panel - Chat */}
        <div className="w-[420px] flex-shrink-0">
          <BriefChatPanel
            messages={messages}
            suggestedQuestions={suggestedQuestions}
            input={input}
            setInput={setInput}
            sendMessage={sendMessage}
            isLoading={isLoading}
          />
        </div>

        {/* Right panel - Brief empty state */}
        <div className="flex-1 overflow-y-auto">
          <BriefEmptyState
            isBriefComplete={isBriefComplete}
            onStartEditing={handleStartEditing}
          />
        </div>
      </div>
    </div>
  );
}
```

**Key decisions:**
- `router.push('/')` navigates to the current editor page per D-12 (no confirmation dialog). In Phase 3, the editor will move to `/editor` and this will be updated — for now, `/` is where the editor lives.
- Header is minimal: just app title "Email Brief" — no undo/redo or export buttons (those belong to the editor). Per Research open question 2 recommendation: include minimal header for visual consistency.
- Layout reuses exact same pattern as `app/page.tsx`: `h-screen w-screen overflow-hidden flex flex-col` with `w-[420px]` left panel and `flex-1` right panel (per BRIEF-01).
- No navigation sidebar on brief page (deferred to Phase 3 per CONTEXT.md deferred ideas).

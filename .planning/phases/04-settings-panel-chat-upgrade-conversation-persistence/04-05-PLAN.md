2. Update `app/api/brief/chat/route.ts` to persist messages:

The current route receives `{ message, conversation_id }` and streams Dify response.

Add a new optional field to the request body: `threadId` (the assistant-ui thread remote ID).

After streaming completes (after the `while(true)` loop ends), if `threadId` is provided:
- Save the user message to the DB: `db.insert(messages).values({ id: crypto.randomUUID(), conversationId: threadId, role: 'user', content: message.trim(), createdAt: new Date().toISOString() })`
- Save the assistant response to the DB: `db.insert(messages).values({ id: crypto.randomUUID(), conversationId: threadId, role: 'assistant', content: fullAssistantContent, createdAt: new Date().toISOString() })`
- If a `difyConversationId` was received from Dify during streaming, update the thread: `db.update(conversations).set({ difyConversationId, updatedAt: new Date().toISOString() }).where(eq(conversations.id, threadId))`

To accumulate the full assistant content during streaming, modify the stream handler to track the full response text. Add a variable `let fullAssistantContent = '';` before the stream starts, and append `parsed.answer` to it in the message/agent_message handler.

Important: The message persistence happens AFTER the stream is fully sent to the client. This is done inside the `finally` block of the ReadableStream's `start` method. Use a closure to capture `threadId`, `message`, and `fullAssistantContent`.

Also: When `threadId` is provided, look up the thread to get its `agentId`, then look up that agent's config to use for the Dify API call. This replaces the `getActiveAgentConfig()` call when a specific thread is being used (the thread is already linked to an agent).

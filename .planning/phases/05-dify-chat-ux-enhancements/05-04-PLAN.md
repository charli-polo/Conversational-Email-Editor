**2. Add parameters context** — The parameters (file_upload, speech_to_text) are fetched in `brief-runtime-provider.tsx` but not passed to brief-thread.tsx. Two options:
- Option A: Create a React context in brief-runtime-provider.tsx to share params
- Option B: Fetch parameters independently in brief-thread.tsx

Use Option A — add to brief-runtime-provider.tsx:
```typescript
export const DifyParamsContext = createContext<{
  speech_to_text: { enabled: boolean };
  file_upload: { image: { enabled: boolean; number_limits: number; transfer_methods: string[] } };
} | null>(null);
export function useDifyParams() { return useContext(DifyParamsContext); }
```

Wrap children in the provider. Then in brief-thread.tsx, import and use `useDifyParams()`.

**Actually**, since the RuntimeHook inside `useRemoteThreadListRuntime` can't easily share state outside via context (it's inside the hook), the cleaner approach is Option B — fetch params directly in brief-thread.tsx using a simple useEffect. The data is small and cacheable.

Add to `BriefThread` component:
```typescript
const [difyParams, setDifyParams] = useState<{
  speech_to_text: { enabled: boolean };
  file_upload: { image: { enabled: boolean; number_limits: number; transfer_methods: string[] } };
} | null>(null);

useEffect(() => {
  fetch(`${basePath}/api/brief/parameters`)
    .then(r => r.json())
    .then(data => { if (!data.error) setDifyParams(data); })
    .catch(() => {});
}, []);

const fileUploadEnabled = difyParams?.file_upload?.image?.enabled ?? false;
const sttEnabled = difyParams?.speech_to_text?.enabled ?? false;
```

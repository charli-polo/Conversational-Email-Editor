    **Step 1: Add `resetDatabase` to test-helpers.ts**

    Add a new exported async function to `__tests__/fixtures/test-helpers.ts` that truncates all tables via the API or directly via SQLite. Since tests run in browser context with API access (Playwright), the cleanest approach is an API-based reset endpoint OR direct SQLite truncation via a helper that runs server-side.

    The simplest approach: use `APIRequestContext` to call DELETE on all conversations (which cascades to messages and conversation_tags), then delete all tags and agents. Add this function after the existing `seedConversation` export:

    ```typescript
    /** Reset test database by deleting all seeded data via API calls */
    export async function resetDatabase(request: APIRequestContext): Promise<void> {
      // Get all conversations and delete them (cascades to messages + conversation_tags)
      const threadsRes = await request.get('/api/threads');
      const threads = await threadsRes.json();
      for (const thread of threads) {
        await request.delete(`/api/threads/${thread.id}`);
      }

      // Get all agents and delete them
      const agentsRes = await request.get('/api/agents');
      const agents = await agentsRes.json();
      for (const agent of agents) {
        await request.delete(`/api/agents/${agent.id}`);
      }

      // Tags with no conversations are orphaned -- delete via direct DB call
      // Since there's no tag DELETE API, use a dedicated cleanup endpoint or
      // accept that orphan tags may remain (they don't affect empty-state tests
      // since empty state checks for conversations, not tags)
    }
    ```

    IMPORTANT: Check if `/api/threads` returns an array directly or a wrapped object. Read the GET handler at `app/api/threads/route.ts` to confirm the response shape. Adjust parsing accordingly.

    If orphan tags cause TAB-03b to fail (because `visibleTabs` derives from conversations, but `allTags` might come from a separate query), we may also need to delete tags. Check if there is a DELETE /api/tags endpoint. If not, the simplest fix is to add one OR to truncate the tags table directly via a test-only API route.

    Alternative approach (if API-based reset is insufficient): Create a `__tests__/reset-db.ts` script that uses `better-sqlite3` directly to run `DELETE FROM conversation_tags; DELETE FROM messages; DELETE FROM conversations; DELETE FROM tags; DELETE FROM test_prompts; DELETE FROM agents;` against `./data/test-db.sqlite`. Then call it from test-helpers via a child process or import. This is more reliable than API-based deletion.

    Choose whichever approach works. The key requirement is: after `resetDatabase()` runs, the DB has zero rows in conversations, messages, conversation_tags, tags, agents, and test_prompts.
